# AI Usage Report

## Tools Used and Use Cases

### ChatGPT / Codex

ChatGPT / Codex was used during Assignment 3 for:

- planning the implementation against the rubric
- identifying which existing features already satisfied earlier assignment requirements
- designing a portfolio-appropriate API integration using the GitHub API
- implementing stronger project filtering and sorting logic
- adding state management with `localStorage`
- improving form validation and helper text behavior
- suggesting performance improvements such as lazy-loaded images
- rewriting and expanding the project documentation

### Prior AI Support from Earlier Assignments

Earlier assignment work also used AI for:

- theme toggle refinement
- project filtering structure
- form validation ideas
- layout and spacing suggestions

Assignment 3 built on top of that previous work and extended the logic significantly.

## Benefits of Using AI

AI assistance helped with:

- turning rubric requirements into a concrete implementation plan
- speeding up debugging when UI or validation behavior was not quite right
- refining wording for helper messages and documentation
- improving consistency between newly added UI elements and the existing design
- reducing trial-and-error in JavaScript feature implementation

## Challenges and Limitations

Some AI-generated or AI-suggested ideas still required manual correction.

Examples:

- helper text initially looked too aggressive and needed tone and styling adjustments
- one validation message did not correctly handle the over-300-character case
- some UI additions worked functionally but needed a later consistency pass to fit the existing design
- documentation drafts needed rewriting so they matched the final implemented feature set exactly

AI was useful for acceleration, but not all suggested output was ready to use directly.

## Learning Outcomes

Using AI in Assignment 3 helped reinforce:

- how to integrate a REST API using `fetch`
- how to combine multiple conditions in front-end logic
- how `localStorage` can be used for lightweight state persistence
- how to build more user-friendly validation feedback
- how to think about performance improvements even in a small static site
- how to review AI output critically instead of accepting it blindly

## Responsible Use and Modifications

All AI-generated suggestions were reviewed and adapted manually before inclusion.

Specific responsible-use practices:

- testing each feature in the browser after implementation
- adjusting validation rules and helper text based on actual UI behavior
- editing generated code to match the structure and style of the project
- rejecting suggestions that were unnecessarily complex for a static portfolio
- rewriting documentation so it accurately reflected the final work

The final submission represents personal understanding, manual review, and direct testing of all AI-assisted work.
